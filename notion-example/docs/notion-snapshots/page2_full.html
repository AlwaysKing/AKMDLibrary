拖拽排序

关键点是三个
可以被拖拽的对象要实现 onDrag
HStack{ } .onDrag { // NSItemProvider 向系统注册的被拖动的内容，因为这个拖拽是系统行为，除了可以拖拽到自己还可以拖拽到其他接收drop的app里面的，那么数据就是通过这个来提供的 return NSItemProvider(object: "\(d)" as NSString) }

接收放置的元素实现onDrag
onDrag 函数要提供接受放置的数据类型，以及一个处理Drag的Delegate
HStack{ } .onDrop(of: [UTType.text], delegate: DropRelocateDelegate(item: d, listData: $data, currentData: $draggingData))

实现Drag的Delegate
struct DropRelocateDelegate: DropDelegate { let item: Int @Binding var listData: [Int] @Binding var currentData: Int? // 当拖放操作完成时调用，用于执行实际的放置操作。你可以在该方法中更新数据模型或执行其他必要的操作。该方法返回一个布尔值，指示是否成功执行放置操作。 func performDrop(info: DropInfo) -> Bool { self.currentData = nil return true } // 当拖放项目进入放置区域时调用。你可以在该方法中执行一些视觉反馈或其他操作。 func dropEntered(info: DropInfo) { // 对移动位置进行判断 if item != currentData { let from = listData.firstIndex(of: currentData!)! let to = listData.firstIndex(of: item)! if listData[to] != currentData { listData.move(fromOffsets: IndexSet(integer: from), toOffset: to > from ? to + 1 : to) } } } // 当拖放项目在放置区域内移动时调用。你可以在该方法中执行一些视觉反馈或其他操作 // 对视觉效果似乎并没有项目影响 func dropUpdated(info: DropInfo) -> DropProposal? { return DropProposal(operation: .move) } // 当拖放项目离开放置区域时调用。你可以在该方法中执行一些清理操作或还原视觉反馈。 func dropExited(info: DropInfo) { self.currentData = nil } // 用于验证拖放操作是否接受指定类型的数据。该方法返回一个布尔值，指示是否接受拖放的数据类型。 func validateDrop(info: DropInfo) -> Bool { return info.hasItemsConforming(to: [UTType.text]) } }
